## Overview

This project simulates a real-time electrical grid monitoring system and builds a production-grade streaming data pipeline entirely inside **Databricks** using the classic **Bronze → Silver → Gold** medallion architecture.

The data source is a **synthetic IoT stream** generated by a PySpark simulator (`000_generate_electrical_data.ipynb`) that mimics 50 smart meters and 5 distribution transformers. The simulator generates realistic voltage, current, power, and energy readings every 30 seconds, and randomly triggers transformer outages with automatic recovery — replicating real-world auto-recloser behaviour on a distribution grid.

Raw streaming data lands in **Delta Lake volumes** under a Unity Catalog landing zone. **Lakeflow Declarative Pipelines** then process the data through Bronze, Silver, and Gold layers — delivering clean, enriched tables ready for AI/BI dashboards, Databricks SQL, and an AI Genie Bot.

---

## Project Goal

- Simulate a live electrical grid IoT stream using PySpark
- Ingest raw meter, transformer, and event data into a Delta Lake landing zone
- Build a documented Bronze → Silver → Gold medallion pipeline using Lakeflow Declarative Pipelines
- Deliver Gold tables optimized to answer operational and analytical questions about grid health
- Serve data to AI/BI dashboards and Databricks SQL for real-time monitoring

## Data Architecture (Medallion)

![Architecture diagram](./img/archi_diagram.gif)

### Medallion Layers & Data Flow

The pipeline follows a clean **Bronze → Silver → Gold** medallion pattern inside Databricks Unity Catalog:

- **Landing Zone**: 3 raw Delta volumes ingested every 30 seconds from the PySpark simulator (`/meters/`, `/transformers/`, `/grid_events/`)
- **Bronze**: Raw data read from landing zone volumes with no transformations — schema enforced, ingestion timestamp preserved
- **Silver**: Cleaned, typed, and enriched Delta tables — nulls handled, outage flags derived, meter-to-transformer relationships resolved
- **Gold**: Business-ready tables for operational monitoring and analytics — aggregated load metrics, outage summaries, meter health profiles

## Source Dataset

Data is generated synthetically by the simulator notebook `000_generate_electrical_data.ipynb`. It writes to the following **Unity Catalog Delta Lake volumes** every 30 seconds:

```
/Volumes/electrical_grid/00_landing_zone/electrical_stream/
├── meters/
├── transformers/
└── grid_events/
```

| Volume | Approx. rows/hour | Description |
|---|---|---|
| `meters/` | ~6,000 | One reading per meter per batch — voltage, current, power, energy |
| `transformers/` | ~600 | One reading per transformer per batch — load, capacity, status |
| `grid_events/` | Variable | Logged only on outage start (`breaker_trip`) or recovery (`breaker_reclose`) |

### Simulator overview

| Parameter | Value | Description |
|---|---|---|
| Smart meters | 50 | 10 meters assigned per transformer |
| Transformers | 5 | 500 kVA rated capacity each |
| Batch interval | 30 sec | How often a new batch of readings is written |
| Outage probability | 3% | Per-batch chance a transformer trips |
| Outage duration | 3 batches (~90 sec) | Transformer auto-recovers after this |
| Event-time latency | Up to 20 sec | Simulated sensor delay behind batch time |

## Gold Layer – Final Tables

The Gold layer delivers **production-ready tables** for dashboards, DBSQL queries, and the AI Genie Bot.

| Table | Description |
|---|---|
| `gold_meter_health` | Per-meter summary — avg voltage, avg power, total energy, outage count, last seen timestamp |
| `gold_transformer_load` | Per-transformer load profile — avg/max load kW, % capacity utilisation, total outage batches |
| `gold_outage_log` | Full outage event history — transformer ID, trip time, reclose time, outage duration in minutes |
| `gold_grid_summary` | Rolling grid-wide KPIs — total active power, total energy, number of offline transformers, timestamp |
| `gold_meter_daily` | Daily energy consumption per meter — useful for trend analysis and anomaly detection |

## How to Reproduce

This project uses a **PySpark simulator** to generate streaming data into Delta Lake, and **Lakeflow Declarative Pipelines** to process it through the medallion layers inside Databricks.

Follow the steps below to recreate the environment.

### 1. Set Up Databricks Workspace

- Go to [https://databricks.com](https://databricks.com) and sign in
- Ensure your workspace has **Unity Catalog** enabled
- You will need access to: Notebooks, Delta Live Tables / Lakeflow, DBSQL, and AI/BI Dashboards

### 2. Create the Unity Catalog Structure

In your Unity Catalog, create the following:

```
Catalog:  electrical_grid
Schema:   00_landing_zone
Volumes:
  └── electrical_stream/
        ├── meters/
        ├── transformers/
        └── grid_events/
```

- Go to **Data** → **Unity Catalog** → create catalog `electrical_grid`
- Inside it, create schema `00_landing_zone`
- Inside the schema, create an **External or Managed Volume** named `electrical_stream`

### 3. Run the Data Generator

- Open `000_generate_electrical_data.ipynb` in your Databricks workspace
- Attach it to a running cluster
- Run all cells — the simulator will start writing batches every 30 seconds and print confirmations:

```
Batch written at 2024-11-01T10:30:00+00:00
```

> Keep this notebook running while you set up the pipeline. Interrupt it when you want to stop data generation.

![Data generator running](./img/generator_notebook.png)

### 4. Create the Lakeflow Declarative Pipeline

- In your workspace → **New** → **Pipeline** (Lakeflow / Delta Live Tables)
- Name the pipeline: `electrical_grid_pipeline`
- Set the pipeline mode to **Continuous** for real-time streaming

Connect the three pipeline notebooks in order:

| Notebook | Layer | Description |
|---|---|---|
| `001_bronze.ipynb` | Bronze | Reads raw Delta from landing zone volumes, enforces schema |
| `002_silver.ipynb` | Silver | Cleans data, derives outage flags, links meters to transformers |
| `003_gold.ipynb` | Gold | Aggregates into business-ready tables |

![Lakeflow pipeline](./img/lakeflow_pipeline.png)

### 5. Configure the Pipeline

In the pipeline settings:

- **Storage location**: point to your Unity Catalog schema (e.g. `electrical_grid.pipeline_storage`)
- **Target schema**: `electrical_grid.gold` (or your preferred Gold schema)
- **Cluster**: use a cluster with Photon enabled for best Delta performance

### 6. Run the Pipeline

- Click **Start** to trigger the pipeline
- Wait for Bronze → Silver → Gold to complete successfully

![Pipeline run](./img/pipeline_run.png)

Verify that all Gold tables have been created in Unity Catalog under your target schema.

### 7. Connect to Databricks SQL

- Open **Databricks SQL** → **Query Editor**
- Select your Gold schema and begin querying:

```sql
SELECT * FROM electrical_grid.gold.gold_transformer_load
ORDER BY avg_load_kw DESC;
```

![DBSQL query](./img/dbsql_query.png)

### 8. Set Up AI/BI Dashboard

- In Databricks → **AI/BI** → **New Dashboard**
- Connect to your Gold tables
- Build visualisations for:
  - Real-time grid load across all transformers
  - Outage frequency and duration trends
  - Meter-level energy consumption
  - Transformer capacity utilisation

![AI/BI Dashboard](./img/dashboard.png)

### 9. AI Genie Bot (Optional)

- In Databricks → **AI/BI** → **Genie**
- Connect Genie to your Gold schema
- Ask natural language questions like:
  - *"Which transformer had the most outages this week?"*
  - *"Show me meters with voltage below 225V in the last hour"*
  - *"What is the total energy consumed today?"*

![AI Genie Bot](./img/genie_bot.png)
